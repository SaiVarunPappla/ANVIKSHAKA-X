# ANVIKSHAKA-X Documentation

This directory contains additional documentation and development notes for the ANVIKSHAKA-X project.

## Directory Structure

### `internal/`
Technical documentation and implementation guides useful for contributors:

- **FRONTEND_UPGRADE_HANDOFF_STATUS.txt** — Comprehensive log of frontend upgrade process (F1-F6 batches)
- **FRONTEND_PREMIUM_UPGRADE_PLAN.md** — UI/UX enhancement plan and design decisions
- **FRONTEND_DUPLICATE_REQUEST_AUDIT.md** — Performance audit identifying duplicate API calls
- **MISSION_FLOW_80S_AUDIT.md** — API optimization audit and improvements

These files provide valuable context for understanding:
- Frontend architecture evolution
- Performance optimization decisions
- UI/UX design system development
- API efficiency improvements

### `archive/`
Historical development notes and changelogs from the implementation process:

- **AI_INTEGRATION_CHANGES.md** — Ollama integration fixes and debugging notes
- **BATCH_1A-1E_*.txt/md** — Implementation changelogs and diffs from various development phases

These files are kept for historical reference but are not required for understanding the current system.

## For Contributors

If you're contributing to ANVIKSHAKA-X:

1. **Start with the main README** in the project root
2. **Review `internal/` docs** to understand design decisions
3. **Check `archive/`** only if investigating historical issues

## For Users

Main documentation is in the root **README.md** file. This `docs/` directory is primarily for developers and contributors.
